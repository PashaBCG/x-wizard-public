# Onboarding Interview — Example Answers

This file helps users answer the skill-building interview questions during X-Wizard onboarding.
It is a fallback reference: if an Excel file was scanned and a schema was found, the agent should
generate tailored questions from the scan instead of using these generic templates.

---

## Excel / Data File

**Which file do you use most?**

Just name it or describe what it tracks — no need to be exact.

> "headcount tracker — one row per person, monthly updates, offices and grades"

> "pipeline report I check every Monday morning"

> "BCGX people directory — names, emails, offices, grades"

---

**Where is it stored?**

Paste the full file path. To copy it quickly:
- **Mac:** Right-click the file in Finder → hold Option → click "Copy ... as Pathname"
- **Windows:** Shift + right-click the file → "Copy as path"

Or just describe where it is — plain English works too:

> "OneDrive, inside a folder called Reports"

> "/Users/me/Documents/Data/headcount_2025.xlsx"

> "Desktop, file called pipeline_tracker.xlsx"

---

**What questions do you usually answer with it?**

Write them like you'd ask a colleague — a few examples is plenty:

> "How many people are active in London this month?"

> "Who joined or left in the last 30 days?"

> "What's our headcount in Boston vs NYC right now?"

> "Show me everyone at grade JC or above in a specific practice."

> "What changed since last month's report?"

---

**What should the output look like?**

> "A short bullet summary I can paste into Slack or a Teams message"

> "A new tab in the same Excel file with the filtered results"

> "An email draft to my manager with the key numbers"

> "A PDF summary I can share with the team"

---

## Routine Emails

**Which email do you send most often?**

One sentence is enough:

> "Weekly project status update to the client team"

> "Monthly headcount report to the practice area leader"

> "Friday pipeline summary to my CD"

> "New joiner welcome email with onboarding links"

---

**Who are the recipients?**

> "Always the same four people — I'll give you their names"

> "Varies by project — I pick from my contacts each time"

> "My manager and the case team lead — always the same"

---

**What stays the same every time?**

> "The subject line, the sign-off, and the overall structure — intro / metrics / next steps"

> "Opening line and signature — the body changes each time"

> "The table format — same columns, just different numbers"

---

**What changes each time — and how do you decide what to put there?**

> "The numbers — I pull them manually from an Excel file each Friday"

> "The project name and key updates — I write those fresh each time"

> "The date and status — everything else stays the same"

---

**Should X-Wizard draft it for review, or send directly?**

> "Draft only — I want to review before it goes"

> "Send directly once it looks right"

---

## Report / Deck Compilation

**What report do you compile most often?**

> "Monthly NCC summary for the practice area — goes to the leadership team"

> "Weekly pipeline update — a short Excel I send to my CD every Friday"

> "Quarterly headcount review deck — PowerPoint, 10–15 slides"

---

**Where does the data come from?**

> "One Excel file — I update it manually each month"

> "Two sources: a shared folder with exports + an email summary I write myself"

> "I pull from Tableau and paste into a Word doc"

---

**What does the output look like?**

> "A PDF summary — one page, key metrics and a short narrative"

> "A PowerPoint deck — pre-structured, I just fill in the numbers"

> "An email with a table of results pasted in"

> "A new Excel tab with the final formatted view"

---

**How often do you do this?**

> "Weekly — every Friday before EOD"

> "Monthly — first working day of the month"

> "Ad hoc — whenever a partner asks for an update"
